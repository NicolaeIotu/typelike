Module: typelikeCustom

typelikeCustom behaves in the same way as typelike, but expects that the last parameter passed to be a special settings object which is mandatory and has the following default format for standard typelike calls:

{
    maxDepth: 0,
    properties: {
        allowMissing: false
    }
}

The comparison is done using the keys and the type of data where applicable.
typelikeCustom iterates as deeply as possible any arrays, objects, maps and sets, while taking into account the settings. For type checking typelikeCustom uses xtypeof which is included as part of the application.

Parameters:
Name Type Description
obj object

A test object to match against the template(s).
templates object

Templates are trusted objects. The test object obj must match in depth the types of these templates.
settings Object

The settings object to be used for this call only.

Returns:

true if the target object obj matches any of the templates provided, or false otherwise

Type
boolean
Example
const { typelikeCustom } = require('typelike')
const testObject = {
    lvl1: { sm: 'type ... like' },
    arr: [[1], 'abcdef']
}
const templateObject = {
    lvl1: { lvl2: [3, 4, 212], sm: '' },
    arr: [[44, ''], '']
}
const settings = {
    maxDepth: 3,
    properties: {
        allowMissing: true
    }
}
console.log(typelikeCustom(testObject, templateObject, settings)) // true
